## 强制执行规则

### ⚠️ 规则校验铁律（最高优先级，2026-06-12 白名单事故后新增）

**任何新规则、新错字、新阈值加入知识库前，必须完成以下三步，不通过不写入：**

| 步骤 | 操作 | 验证标准 |
|:--:|------|------|
| 1 | **法规溯源** | 每条新规则必须能引用到具体的 GB 条款号（如 GB7718-4.2.1），不能靠"常识""经验""我记得" |
| 2 | **逐条验算** | 涉及数值或列举清单的（如阈值表、白名单、禁用词库），必须逐条对照法规原文验算，发现任意一条不合规 → 整个规则方案废弃重来 |
| 3 | **历史回测** | 新规则用最近 3 份已审核标签回测——会不会在合规标签上误报？误报率>0 → 规则不入库，先修正 |

> 🚫 白名单事故教训：2026-06-12 提出的"合法短配料白名单"13 项中 7 项不合规（盐/糖/油/酱/蜜/茶/奶/酒均不可单字使用）。如未及时发现，后续所有审核会在"盐""糖"等违规单字上产生系统性漏判。
> 
> **两条底线，不矛盾：**
> - **审核执行底线**：宁可多报不可漏，可疑处标🟡而非跳过。🟡的代价是人工多看一眼，漏判的代价是标签带病出厂。
> - **知识库写入底线**：不确定不入库，入必准确。假规则入库会造成不可逆的系统性漏判，比单次漏判严重得多。

### 审核流程（必须执行，不可跳过）
1. 分辨率检测（<1500px直接拦截）
2. 关键字段完整性（缺SC+配料+营养表→拦截）
3. 配料表项数合理性（<5项→提醒）
4. 双通道OCR交叉验证（镜像层不一致只🟡不🔴）
5. 分块OCR（>4MP时10%重叠分块）
6. 输出只给PNG标注稿（不给MD）
7. 输出格式：600dpi、左上角标注文件类型/OCR分辨率

### 安全限制
- 附件/图片/证书处理完毕立即删除临时文件（test_bc.jpg等）
- 不保留原始设计稿副本（只存提取后的文字+审核报告）
- 不联网查条码注册状态（编码中心无公开API）
- OCR漏扫致敏原/配料时🟡提醒，不判🔴
- 镜像层文字差异不自动判错
- 低分辨率(<2500px)条码异常只🟡不🔴
- JPG版审核的数值类问题（能量/NRV/碳水偏差）必须以AI文件或定稿文案复核，不基于JPG OCR下结论

### 切换逻辑（自动）

```
OCR调用前检查：
    ├── 返回 error_code=17（日限额）或 error_code=18（QPS超额）
    ├── 当前账户 A → 自动切换到 B
    ├── 当前账户 B → 报🔴 "双账户日配额均已耗尽，建议等待明日0点重置"
    └── 切换后记录日志：{时间} 账户{A}→{B}，原因{error_msg}
```

> 🔄 切换仅在当前调用失败时触发，不跨天记忆（每天从账户A开始）。

### 百度OCR配额速查

| 端点 | 日配额 | 特点 |
|------|:--:|------|
| accurate_basic | 500次(日/账户) / 1000次(月免费/账户) | 高精度，审核专用 |
| general_basic | 500次(日/账户) | 低精度，仅用于简单场景 |
| 超额单价 | 0.004元/次(accurate) | 超出后自动扣费 |

> ⚠️ 双账户合计月免费配额2000次。缩图至3.9MP（1次/张）为首选策略，月产能上限2000批。

### 两级OCR策略（防配额浪费）

```
设计稿 → 缩图至3.9MP → accurate_basic（1次）
    ↓
三道质检（分辨率/关键字段/配料项数）
    ↓
┌─ 质检通过 → 直接进入审核
└─ 质检不通过（文字过小/关键字段缺失）→ 仅此单张回退原图分块OCR（~15次）
```

> 🔴 严禁使用 general_basic 作为审核数据源（精度不足，宁可报"OCR失败"）。
> 🔴 分块数 > 50 → 中断报错（分块公式异常，如2026-06-02的330×2=660块bug）。
> 🟡 单次审计前检查余额 `curl "https://aip.baidubce.com/rest/2.0/ocr/v1/accurate_basic?access_token={token}"` 或查看控制台。余额 < 100 时提醒用户。

### 分块计算公式（防复发）

```python
MAX_PX = 3_900_000  # 3.9MP，留100K余量
aspect = w / h
tile_h = int(sqrt(MAX_PX / aspect))
tile_w = int(MAX_PX / tile_h)
cols = ceil(w / tile_w)   # 不用stride公式！
rows = ceil(h / tile_h)
# 单图调用次数 = cols × rows，应 ≤ 25
if cols * rows > 50:
    raise QuotaError(f"分块数异常({cols}x{rows}={cols*rows})，中断防配额浪费")
```
